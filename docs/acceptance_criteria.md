# Job Search RSS Module Acceptance Criteria

## WBS 1.1 MVPシナリオ一覧

### AC-001: 購読条件を登録できる

- 利用者は地域、職種、または地域と職種の組み合わせを購読条件として登録できる。
- 登録時には、サイト固有パラメータではなく共通モデルの地域・職種として受け付ける。
- 登録された購読条件は正規化され、同じ意味の条件を同一条件として扱える。
- 登録結果として、後続の収集条件導出とRSS生成で参照できる購読条件IDを得られる。

### AC-002: 購読条件から収集条件を導出できる

- 登録済みの購読条件から、atGPへアクセスする単位の収集条件を導出できる。
- 同一の正規化済み条件が複数登録されても、収集条件は重複しない。
- 収集条件は購読条件とは分離され、将来の収集頻度最適化に備えて独立して扱える。

### AC-003: 収集条件に基づいて求人を収集できる

- 収集処理は登録済みの収集条件だけを対象にする。
- atGPアダプタは地域条件、職種条件、地域と職種の組み合わせ条件で求人一覧を取得できる。
- 求人一覧と求人詳細は、RSS配信と差分検出に必要な共通求人モデルへ変換される。
- 収集結果には、どの収集条件で求人が見えていたかを記録できる。

### AC-004: 求人の新規・更新・削除を検出できる

- 前回スナップショットに存在しない求人は、新規変更として記録される。
- 同一求人の内容ハッシュが変わった場合だけ、更新変更として記録される。
- 同一求人を再収集して内容が変わらない場合、不要な変更履歴は追加されない。
- 正常に収集が完了した条件で、前回見えていた求人が今回見えない場合だけ、削除または掲載終了として記録される。

### AC-005: RSSを生成できる

- RSS生成は保存済みの変更履歴だけを参照する。
- RSS生成時にatGP、HTTPクライアント、Playwrightなどの外部アクセス処理は呼び出されない。
- RSS項目にはタイトル、リンク、説明、GUID、公開日時、変更種別、地域、職種を含められる。
- 利用者は地域、職種、変更種別、購読条件に基づいて変更履歴を購読できる。

### AC-006: 失敗した収集を安全に扱える

- atGPへのアクセス失敗、パース失敗、途中ページ失敗は収集失敗として記録される。
- 収集失敗時は不完全な結果をもとに削除検知を実行しない。
- 失敗した収集は実行履歴として参照でき、求人状態を過剰に変更しない。

### AC-007: HTTPまたは手動入口からMVPフローを実行できる

- FastAPI経由で購読条件を登録できる。
- FastAPI経由で購読条件に対応するRSSを取得できる。
- 運用確認用に、購読条件登録、マスター同期、手動収集をCLIから実行できる。
- 定期収集ジョブは登録済みの収集条件を巡回して実行できる。
